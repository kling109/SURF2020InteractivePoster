Analyzing Error in Fundamental Two-Qubit Quantum Gates

Trevor Kling

Project Introduction

Quantum Computation is an emerging field at the intersection of physics, mathematics, and computer science that has the potential to substantially improve the computational capabilities of scientists, doctors, and others who deal with computationally challenging problems. However, to build a scalable quantum computer, components and operations which are highly accurate are required. In this project, an analysis of one such promising component, the cross-resonance gate, is presented. The cross-resonance gate is a fundamental two-qubit entangling operation for the transmon architecture of quantum computation, which is a promising system for the production of large-scale near-term quantum computers. This project models the cross-resonance gate with high physical accuracy in order to better understand the functionality of the gate and how to better improve its fidelity.

Qubit Basics

Qubit Properties
  • Classical Bits: 0 or 1
  • Qubits: Any superposition of 0 and 1. A superposition is a probabilistic mixture of the chosen basic states (in this case, 0 or 1). A measurement causes this mixture to collapse to one of the basic states.
  • Qubits can also become entangled, which allows the value of one qubit to dictate the value of another qubit when measured.
  • Qubits are often represented as unit spheres, and each point on this sphere represents a valid state for the qubit.

Transmon Circuits Basics

A transmon is a simple quantum circuit which can be used to implement a qubit. Functionally, it is a superconducting quantum oscillator.
Important Properties
  • Anharmonicity: A measure of how much the transmon deviates from the behavior of a harmonic oscillator, like a mass on a spring.
  • Base frequency: The base rate of oscillation. Multiple transmons in the same system are usually detuned from each other by having varying base frequencies.

Formal Simulations

Ideal Gate Operations

An "Ideal" Cross-Resonance gate can be described as a rotation of the control qubit about the "Z" axis, and a rotation of the target qubit about the "X" axis. The magnitude of these rotations is determined by the state of the control qubit; this produces a conditioned behavior on the target qubit.

Hamiltonian Formulation

While the ideal model provides clarity about how the gate acts, it does not provide any insight into how this can be achieved in the laboratory setting. By using a mathematical model called the Hamiltonian, all the components which make up the two transmons and the action being performed can be modeled. This approach is accurate to the real behavior of the system, and tuning its parameters allows for the production of more accurate cross-resonance gates.

Charge Dephasing and Pulse Envelopes

A variety of effects can result in the accuracy of operations degrading over time. One such effect is fluctuations in the electrical charge being passed through the quantum circuit. This can be modeled as a time-dependent shift of the base frequency of the transmon. However, since the operation of the cross-resonance gate is also dependent on the base frequencies of these transmons, this can produce errors in the procedure.
In addition,, typically one cannot go from 0 drive to maximum drive instantly without causing unwanted errors. To avoid this, pulse envelopes are used to smooth out the process of applying a drive. These envelopes allow the drive to smoothly transition from 0 to maximum drive, then back to 0. For all prior simulations, a cosine ramp was used.

Cross-Resonance Gate Basics

The Cross-Resonance Gate is an operation performed on 2 transmons. In general, it causes the target transmon to rotate based on the state of the control transmon. This is achieved by causing the two transmons to interact, then driving the control transmon at the same base frequency as the target transmon.

Figure 3: A classical CR-Gate counterpart [3]

A classical analogy to this system is two pendulums coupled by a spring and driven with some periodic force. The is applied to only the first pendulum, and at the frequency of the second pendulum. Due to the spring-based interaction, this results in an increase in amplitude for the swinging of the second pendulum, while the effects on the first pendulum are off-resonance and thus average to 0 over many cycles.
